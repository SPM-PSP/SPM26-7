# 一、模块一：健康数据管理核心

## 1.1 开发模式选择

### 1.1.1 前端架构模式

| 模式 | 选择 | 应用场景 |
|------|------|---------|
| 组件模式 | **Client Component** (`"use client"`) | Dashboard 页面及所有交互组件均为客户端组件，因为依赖浏览器 API（IndexedDB/localStorage/Canvas） |
| 状态管理模式 | **React useState + useEffect** 驱动，无外部状态库 | Dashboard 使用 10+ useState + 4 useEffect 管理复杂业务状态 |
| 表单模式 | **受控组件 (Controlled Component)** | 全部 input/textarea/select 通过 value + onChange 管理 |
| 数据获取模式 | **自定义 Hooks 封装** | useIndexedDB / useLocalStorage / useDateRecords / useExportReminder 封装数据层 |
| 数据持久化 | **三层存储**（localStorage + IndexedDB + 内存 React State） | 配置类数据存 localStorage，业务流水存 IndexedDB，当前视图状态存 React State |
| 条件渲染 | **状态驱动** | `isProcessing`/`isLoading`/`isCompressing` 控制加载态，`dailyLog.foodEntries.length === 0` 控制空状态 |
| 国际化 | **next-intl useTranslations** | 所有用户可见文本均通过 `t('key')` 翻译函数获取 |

### 1.1.2 数据流设计

```
用户操作 (UI Event)
    │
    ▼
事件处理函数 (handleXxx)
    │
    ├─→ setState (更新 React State) ──→ 组件重渲染
    │
    ├─→ saveDailyLog() ──→ IndexedDB.put()
    │
    └─→ fetch("/api/openai/parse") ──→ AI 解析 ──→ result JSON
            │
            ▼
        setDailyLog(updatedLog) ──→ saveDailyLog() ──→ recalculateSummary()
            │
            ▼
        refreshRecords() ──→ setChartRefreshTrigger(+1)
```

**核心原则**：单向数据流，React State 为唯一数据源（Single Source of Truth），IndexedDB 为持久化副本。

### 1.1.3 组件通信模式

```
Dashboard (page.tsx)                     ← 数据拥有者 & 状态管理中心
    │
    ├──→ Calendar (shadcn)              ← props: selectedDate, onSelect, hasRecord
    ├──→ ManagementCharts               ← props: selectedDate, refreshTrigger
    ├──→ DailyStatusCard                ← props: date, initialStatus, onSave
    ├──→ FoodEntryCard[]                ← props: entry, onDelete, onUpdate
    ├──→ ExerciseEntryCard[]            ← props: entry, onDelete, onUpdate
    ├──→ DailySummary                   ← props: summary, calculatedBMR, calculatedTDEE, tefAnalysis
    └──→ SmartSuggestions               ← props: suggestions, isLoading, onRefresh
```

**通信方式**：
- 父→子：**Props 传递**（数据 + 回调函数）
- 子→父：**回调函数**（`onDelete(id)`, `onUpdate(entry)`, `onSave(status)`）
- 跨组件共享：通过 Dashboard 集中持有状态并下发

## 1.3 数据持久化层设计

### 1.3.1 useIndexedDB Hook

```
┌───────────────────────────────────────────────────┐
│              useIndexedDB(storeName)               │
├───────────────────────────────────────────────────┤
│ 状态                                               │
│   db: IDBDatabase | null                          │
│   isLoading: boolean                               │
│   isInitializing: boolean (初始为 true)            │
│   error: Error | null                              │
├───────────────────────────────────────────────────┤
│ 初始化 (useEffect)                                 │
│   indexedDB.open("healthApp", 2)                   │
│     onupgradeneeded:                               │
│       若 storeName 不存在 → db.createObjectStore() │
│       若 "aiMemories" 不存在 → db.createObjectStore() │
│     onsuccess: db = result, isInitializing = false │
│     onerror: setError                             │
├───────────────────────────────────────────────────┤
│ API                                                │
│   getData(key)    → Promise<any>                   │
│     内部: store.get(key)  事务模式 readonly        │
│   saveData(key, data) → Promise<void>              │
│     内部: store.put(data, key) 事务模式 readwrite  │
│   deleteData(key) → Promise<void>                  │
│     内部: store.delete(key)                        │
│   clearAllData() → Promise<void>                   │
│     内部: store.clear()                            │
└───────────────────────────────────────────────────┘
```

**关键设计决策**：
- 数据库版本固定为 **2**，`onupgradeneeded` 中按需创建 ObjectStore（幂等）
- 所有操作通过 `useCallback` + `db` 依赖优化，避免不必要的重渲染
- 组件卸载时 `db.close()` 防止连接泄漏
- `put(data, key)` 语义：键相同时覆盖（支持更新）

### 1.3.2 useLocalStorage Hook

```
签名: useLocalStorage<T>(key, initialValue) → [storedValue, setValue]

实现:
  初始化: lazy useState
    若 SSR (typeof window === "undefined") → 返回 initialValue
    否则:
      try { JSON.parse(localStorage.getItem(key)) } catch { initialValue }

  setValue(value: T):
    1. 支持函数式更新: value instanceof Function ? value(storedValue) : value
    2. setStoredValue(valueToStore)  ← 更新 React State
    3. localStorage.setItem(key, JSON.stringify(valueToStore)) ← 同步持久化
```

**关键设计决策**：
- 使用 **lazy initializer**（`() => {...}`）而非直接传值，避免每次都读 localStorage
- `setValue` 用 `useCallback` 包装，依赖 `[key, storedValue]`
- JSON 解析失败时降级返回 `initialValue`，不抛异常阻塞渲染

### 1.3.3 useDateRecords Hook

```
签名: useDateRecords() → { hasRecord(date) → boolean, isLoading, refreshRecords() }

实现:
  loadRecordedDates():
    1. indexedDB.open("healthApp", 2)
    2. healthLogs.getAll() — 获取全部记录
    3. 筛选有数据的条目:
       条件: foodEntries.length>0 || exerciseEntries.length>0 
             || weight!==undefined || dailyStatus || calculatedBMR 
             || calculatedTDEE || tefAnalysis
    4. 提取 date → Set<string>
    5. 设置 recordedDates

  hasRecord(date): recordedDates.has(format(date, "yyyy-MM-dd"))

  refreshRecords(): 调用 loadRecordedDates() 刷新集合
```

**关键设计决策**：
- calendar 组件的 `hasRecord` prop 接收此函数，用于渲染日期标记点
- 初始化时自动加载（useEffect with empty deps），增删改操作后手动 `refreshRecords()`

### 1.3.4 useExportReminder Hook

```
签名: useExportReminder() → ExportReminderState

状态:
  shouldRemind: boolean        // 是否需要提醒
  daysSinceLastExport: number   // 距离上次导出天数
  lastExportDate: Date | null   // 上次导出时间
  hasEnoughData: boolean        // 是否有足够数据
  dataSpanDays: number          // 数据跨度天数

判定逻辑:
  1. checkDataSpan() → 遍历 IndexedDB healthLogs，计算最早→最晚日期跨度
     条件: hasData ← 跨度 >= 2天
  
  2. 无足够数据 → shouldRemind = false
  
  3. 无 lastExportTime → shouldRemind = true (从未导出)
  
  4. 有 lastExportTime → daysDiff = (now - lastExportTime) / (86400000)
     条件: daysDiff >= 2 → shouldRemind = true

定时检查: setInterval(60000*60) 每小时检查一次
```

### 1.4.2 核心业务流程

**流程 1：日期切换加载数据**

```
用户点击日历日期
  → setSelectedDate(date)
  → useEffect 触发（依赖 selectedDate）
    → dateKey = format(selectedDate, "yyyy-MM-dd")
    → getDailyLog(dateKey)  // IndexedDB 查询
      ├── 有数据 → setDailyLog(data)
      │           setCurrentDayWeight(data.weight)
      │           setCurrentDayActivityLevelForSelect(data.activityLevel)
      └── 无数据 → setDailyLog(空 DailyLog)
                   setCurrentDayWeight("")
                   setCurrentDayActivityLevelForSelect(默认值)
```

**流程 2：提交食物/运动记录**

```
用户输入文本 (+ 可选图片) + 点击提交
  → checkAIConfig() 验证 AI 配置完整
  → setIsProcessing(true)
  ├── 有图片 → fetch("/api/openai/parse-with-images", FormData)
  └── 无图片 → fetch("/api/openai/parse", JSON {text, type, userWeight})
  ↓
  接收 result JSON
  → updatedLog = { ...dailyLog }
  → result.food → updatedLog.foodEntries.push(...)
  → result.exercise → updatedLog.exerciseEntries.push(...)
  → recalculateSummary(updatedLog)
  → setDailyLog(updatedLog)
  → saveDailyLog(updatedLog.date, updatedLog)
  → refreshRecords() + setChartRefreshTrigger(prev => prev + 1)
  → 清空 inputText + uploadedImages
  → setIsProcessing(false)
```

**流程 3：删除条目**

```
用户点击条目删除按钮
  → updatedLog = { ...dailyLog }
  → type==="food" → filter foodEntries by log_id
  → type==="exercise" → filter exerciseEntries by log_id
  → recalculateSummary(updatedLog)
  → setDailyLog(updatedLog) + saveDailyLog
  → refreshRecords() + setChartRefreshTrigger(+1)
```

**流程 4：recalculateSummary 算法**

```typescript
// 聚合所有食物条目的总营养成分
foodEntries.forEach(entry => {
  totalCaloriesConsumed += entry.total_nutritional_info_consumed.calories
  totalCarbs += entry.total_nutritional_info_consumed.carbohydrates
  totalProtein += entry.total_nutritional_info_consumed.protein
  totalFat += entry.total_nutritional_info_consumed.fat
  // 微量营养素（非 macros 的字段）聚合到 micronutrients
})

// 聚合所有运动条目的卡路里消耗
exerciseEntries.forEach(entry => {
  totalCaloriesBurned += entry.calories_burned_estimated
})
```

```
│                    ? currentDayData.weight 
│                    : userProfile.weight

└── 5. 返回 { bmr, tdee, tefEnhancement? }
```

## 2.3 TEF 计算引擎

### 2.3.1 基础 TEF 计算公式

```
TEF 总 = Σ (各食物中各宏量营养素的卡路里 × 热效应系数)

蛋白质卡路里 = protein(g) × 4 kcal/g  →  × 0.25 (25%热效应)
碳水卡路里   = carbs(g) × 4 kcal/g    →  × 0.08 (8%热效应)
脂肪卡路里   = fat(g) × 9 kcal/g      →  × 0.02 (2%热效应)

TEF_total = proteinCal × 0.25 + carbsCal × 0.08 + fatCal × 0.02
TEF_percentage = (TEF_total / totalCalories) × 100%
```

**实现细节**（`calculateBaseTEF`）：
```typescript
// 遍历每个食物条目，从 total_nutritional_info_consumed 取值
foodEntries.forEach(entry => {
  const proteinCal = (nutrition.protein || 0) * 4
  const carbsCal = (nutrition.carbohydrates || 0) * 4
  const fatCal = (nutrition.fat || 0) * 9
  
  totalProteinTEF += proteinCal * 0.25
  totalCarbsTEF += carbsCal * 0.08
  totalFatTEF += fatCal * 0.02
  totalCalories += nutrition.calories || 0
})
```

### 2.3.2 TEF 时间衰减模型

```
衰减函数（指数衰减）:
  peakTime = 1.5h          ← TEF 在进食后 1.5 小时达到峰值
  halfLife = 2h            ← 半衰期 2 小时
  
  上升阶段 (0 → 1.5h): factor = hoursElapsed / 1.5
  衰减阶段 (1.5h → 6h): factor = e^(-(hoursElapsed-1.5) × ln(2) / 2)
  消失 (> 6h): factor = 0

currentTEF = Σ (各食物的基础TEF × 时间衰减因子)
```

### 2.3.3 TEF 增强因子自动识别

```
识别流程:
  foodEntries.forEach(entry → entry.food_name.toLowerCase())
    │
    ├── 匹配 caffeine 关键词 (咖啡/茶/红茶/乌龙茶/可乐/能量饮料...)
    │     → 推入 "咖啡因" × 1.10
    │
    ├── 匹配 greenTea 关键词 (绿茶/抹茶/龙井/碧螺春...)
    │     → 推入 "绿茶儿茶素" × 1.12（覆盖 caffeine）
    │
    ├── 匹配 spicy 关键词 (辣椒/胡椒/芥末/生姜/川菜/湘菜/咖喱...)
    │     → 推入 "辛辣食物" × 1.08
    │
    ├── 匹配 cold 关键词 (冰水/冰饮/冰咖啡/冰茶/冰淇淋...)
    │     → 推入 "冷饮热效应" × 1.03
    │
    └── 匹配 metabolicEnhancers (肉桂/姜黄/柠檬/柚子/MCT油...)
          → 推入 "代谢增强物质" × 1.05
```

## 2.4 TEF 缓存系统

### 2.4.1 缓存架构

```
┌─────────────────────────────────────────────────────┐
│              TEFCacheManager (单例)                   │
├─────────────────────────────────────────────────────┤

│    4. JSON.stringify → hash                          │
├─────────────────────────────────────────────────────┤

│    2. 生成 hash                                      │

├─────────────────────────────────────────────────────┤
│  setCachedAnalysis(entries, analysis)                │
│    1. 空 entries → 直接返回                           │
│    2. Map.set(hash, {analysis, timestamp: now})      │
```

### 2.4.2 哈希生成策略

```typescript
  const stableData = foodEntries
    .map(entry => ({
      name: entry.food_name.trim().toLowerCase(),      // 大小写不敏感
      grams: Math.round(entry.consumed_grams * 100) / 100, // 2位小数
      mealType: entry.meal_type,
      nutrition: {
        calories: Math.round((...total_calories) * 100) / 100,
        protein: Math.round(...),
        carbohydrates: Math.round(...),
        fat: Math.round(...),
      }
    }))
    .sort((a, b) => a.name.localeCompare(b.name) || a.grams - b.grams)
    // ^ 先按名称排序，再按重量排序，确保同集合产生相同哈希
  
  return JSON.stringify(stableData)
}
```

**排除字段**：`log_id`（每次新UUID）、`timestamp`（时间无关营养分析）、`is_estimated`（与分析无关）

## 2.7 Dashboard 中 TEF 分析联动

```
Dashboard 中 TEF 相关逻辑 (page.tsx useEffect):

useEffect 监听 [dailyLog.foodEntries, aiConfig, saveDailyLog]:
  │
  ├── 1. 计算当前哈希
  │     currentHash = tefCacheManager.generateFoodEntriesHash(dailyLog.foodEntries)
  │
  ├── 2. 检查缓存
  │     cachedAnalysis = tefCacheManager.getCachedAnalysis(dailyLog.foodEntries)
  │     有缓存 + 当前 dailyLog.tefAnalysis 不同于缓存?
  │       → 直接应用缓存结果，更新 dailyLog，保存 IndexedDB
  │       → 保存 currentHash 到 ref
  │       → return (无需发起 AI 请求)
  │
  ├── 3. 判断是否需要分析
  │     shouldAnalyzeTEF(foodEntries, previousHash) → false?
  │       → return (哈希未变或已有缓存)
  │
  ├── 4. foodEntries.length > 0?
  │     ├── Yes: 启动 15 秒倒计时
  │     │          setTEFAnalysisCountdown(15)
  │     │          每秒: setTEFAnalysisCountdown(prev => prev - 1)
  │     │          15 秒后: performTEFAnalysis(foodEntries)
  │     │            → fetch("/api/openai/tef-analysis", {foodEntries})
  │     │            → 拿 AI 返回的 enhancementMultiplier
  │     │            → generateTEFAnalysis(foodEntries, aiResult.enhancementMultiplier)
  │     │            → 合并 factors（优先 AI 结果，回退本地识别）
  │     │            → tefCacheManager.setCachedAnalysis(foodEntries, finalAnalysis)
  │     │            → 更新 dailyLog.tefAnalysis + saveDailyLog
  │     │
  │     └── No: 清除 TEF 和倒计时
  │
  └── 5. 清理函数: 组件卸载时清除所有定时器
```

**设计要点**：
- **15 秒防抖**：用户连续添加食物时不会频繁触发 AI 请求，15 秒内无新增操作才发起
- **缓存优先**：相同食物组合直接复用缓存，无需等待 AI
- **ref 追踪哈希**：`previousFoodEntriesHashRef` 避免同一组食物重复触发
- **倒计时 UI**：`tefAnalysisCountdown` 显示在 DailySummary 组件中，用户可感知分析进度
