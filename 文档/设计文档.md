# Snapifit-AI 设计文档

## 1. 概述

### 1.1 项目背景
Snapifit-AI 是一个综合性的健康管理和AI辅助系统，旨在通过先进的人工智能技术为用户提供个性化的健康建议、营养指导、运动计划和心理健康支持。系统整合了食物记录、运动追踪、健康数据分析、AI聊天机器人等多种功能，为用户提供全方位的健康管理解决方案。

### 1.2 系统定位
- **健康管理平台**：提供全面的健康数据记录和分析
- **AI辅助系统**：通过AI技术提供个性化建议和指导
- **用户交互中心**：建立用户与AI之间的智能对话系统
- **数据驱动平台**：基于用户数据提供精准的健康建议

### 1.3 核心价值
- **个性化**：根据用户历史数据提供定制化建议
- **智能化**：利用AI技术优化健康管理流程
- **全面性**：覆盖营养、运动、心理等多个健康领域
- **易用性**：简洁直观的用户界面和交互体验

#### 3.9.2 数据关系设计
- **用户表（user_profiles）** 与 **体重记录表（weight_records）**、**活动水平记录表（activity_level_records）**、**食物记录表（food_entries）**、**运动记录表（exercise_entries）**、**专家对话表（chat_{expert_role}）**、**建议历史表（suggestion_history）** 之间存在一对多的关系，通过 user_id 字段进行关联。

#### 3.9.3 数据库性能优化

- **索引策略**：在 user_id、date 等常用查询字段上创建索引

- **查询分析**：定期分析慢查询日志
- **查询计划**：使用EXPLAIN分析查询执行计划

##### 3.9.3.2 存储优化
- **数据分区**：按日期对历史数据进行分区
- **数据压缩**：启用MySQL表压缩
- **数据归档**：定期归档历史数据
- **数据清理**：自动清理过期数据
- **存储分层**：热数据、温数据分离存储

## 8. 细节设计 - AI Memory系统

### 8.1 系统概述
AI Memory系统是Snapifit-AI的核心组件之一，负责存储和管理AI与用户交互过程中积累的知识和记忆。该系统采用IndexedDB作为持久化存储，支持多专家角色的记忆管理，确保AI能够记住用户的偏好、历史交互和个性化信息。

### 8.2 数据模型设计

#### 8.2.1 AIMemory 接口
```typescript
export interface AIMemory {
  expertId: string       // 专家角色ID (如: general, nutrition, fitness)
  content: string        // 记忆内容，限制500字
  lastUpdated: string    // 最后更新时间 (ISO格式)
```
