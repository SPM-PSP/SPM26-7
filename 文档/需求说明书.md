#  菲特勒斯——你的个人健康管理专家需求规格说明书

## 1. 项目概述

菲特勒斯——你的个人健康管理专家 是一个基于AI驱动的个人健康管理工具，旨在通过快速记录、系统分析和智能建议，为用户提供个性化的健康管理和健身指导。该项目采用Next.js 15框架开发，集成了AI API进行智能解析和建议生成。

## 2. 用户需求分析

### 2.1 核心用户群体
- **个人健身爱好者**：需要记录和分析日常饮食、运动数据
- **健康管理用户**：关注体重、营养摄入和健康指标
- **AI技术用户**：希望获得智能化的健康建议和指导

### 2.2 用户痛点
- 传统健身追踪需要繁琐的手动数据录入
- 难以保持数据一致性和获得有意义的洞察
- 缺乏个性化的健康建议和指导
- 多维度健康数据（饮食、运动、心情、睡眠）难以整合分析

## 3. 功能需求

### 3.1 健康记录功能

#### 3.1.1 饮食记录
- **自然语言输入**：支持文本描述食物摄入
- **图片识别**：通过上传食物照片进行自动识别
- **营养成分解析**：自动解析食物的营养成分（卡路里、蛋白质、碳水化合物、脂肪等）
- **餐次分类**：自动识别早餐、午餐、晚餐、零食等餐次类型
- **摄入量记录**：记录食物消耗的克数

#### 3.1.2 运动记录
- **运动类型记录**：支持有氧运动、力量训练、柔韧性训练等
- **运动时长记录**：记录运动持续时间
- **卡路里消耗计算**：基于用户体重和运动类型计算消耗的卡路里
- **运动强度分析**：分析运动的MET值和肌肉群锻炼

#### 3.1.3 每日状态记录
- **心情记录**：记录每日心情状态（1-6级）
- **压力水平**：记录压力程度（1-6级）
- **健康状况**：记录整体健康感受（1-6级）
- **睡眠质量**：记录睡眠质量和时长
- **补充说明**：支持文字描述详细情况

### 3.2 数据分析功能

#### 3.2.1 营养分析
- **总卡路里摄入**：计算每日总卡路里摄入量
- **宏量营养素分析**：分析碳水化合物、蛋白质、脂肪的摄入比例
- **微量营养素分析**：记录其他营养素摄入情况
- **TEF分析**：食物热效应分析，计算消化食物消耗的能量

#### 3.2.2 代谢计算
- **BMR计算**：基础代谢率计算
- **TDEE计算**：每日总能量消耗计算
- **体重变化趋势**：分析体重变化趋势
- **活动水平调整**：根据活动水平调整代谢计算

### 3.3 AI智能建议

#### 3.3.1 智能建议生成
- **基于历史数据分析**：根据7天历史数据生成建议
- **多维度分析**：结合饮食、运动、心情、睡眠等多维度数据
- **个性化建议**：根据用户目标和身体状况提供个性化建议
- **建议分类**：按优先级（高、中、低）分类建议

#### 3.3.2 AI角色系统
- **营养师角色**：提供营养建议
- **健身教练角色**：提供运动建议
- **健康顾问角色**：提供综合健康建议
- **动态记忆**：AI能够记住用户的健康信息和偏好

### 3.4 用户配置功能

#### 3.4.1 个人信息设置
- **基本身体信息**：体重、身高、年龄、性别
- **活动水平**：选择日常活动水平（久坐、轻度、中度、活跃、非常活跃）
- **健康目标**：维持、减重、增重等目标
- **BMR计算公式**：支持Mifflin-St Jeor和Harris-Benedict公式

#### 3.4.2 专业模式
- **健康档案**：记录疾病史、过敏史、药物使用情况
- **生活方式**：记录饮食偏好、睡眠习惯、压力水平
- **健康认知**：记录用户对健康的认知和期望

### 3.5 数据可视化

#### 3.5.1 图表展示
- **营养摄入趋势**：展示每日营养摄入变化
- **体重变化趋势**：展示体重变化曲线
- **卡路里平衡**：展示摄入与消耗的平衡情况
- **TEF分析图表**：展示食物热效应分析结果

#### 3.5.2 数据导出
- **数据备份**：支持数据导出和备份
- **导出提醒**：定期提醒用户备份数据

## 4. 非功能需求

### 4.1 性能需求
- **响应时间**：API 响应时间 ≤ 2 秒。
  · 健康日志提交、数据同步操作 ≤ 1 秒。
  · 净卡路里计算、AI 建议生成及趋势分析报告加载 ≤ 3 秒。
- **数据处理**：支持单用户日均 50+ 条日志记录（含文字、图片）的高效存储与检索。
  · 长期数据（≥ 1 年）分析时可视化报告生成时间 ≤ 5 秒。
  · 同时在线用户数 ≥ 1000 时，系统无明显卡顿或延迟。
- **图片处理**：图片压缩和上传处理时间 ≤ 5 秒。
  · 截图生成时间 ≤ 2 秒，且支持高分辨率（≥ 1080P）输出无失真。

### 4.2 安全需求
- **数据加密**：用户健康日志、深度档案等本地存储数据采用 AES-256 加密。
  · 敏感数据（如 API 密钥）加密存储，不得明文留存。
- **用户隐私**：保护用户个人健康数据，未经授权不得访问或外传。
  · 数据传输全程使用 HTTPS 协议，防止中间人攻击。
- **API安全**：使用 HTTPS 及适当的认证机制（如 JWT 或 OAuth 2.0），确保接口调用身份合法。

### 4.3 用户体验需求
- **响应式设计**：在适配的硬件设备（桌面、手机、平板）上，数据展示、页面截图功能加载流畅，无明显卡顿。
- **国际化支持**：支持中英文界面即时切换，切换操作响应时间 ≤ 0.5 秒。
- **暗色/亮色主题**：支持主题切换，切换过程动画流畅，主题加载时间 ≤ 0.3 秒。
- **动画效果**：交互动画和过渡效果帧率 ≥ 60fps，确保视觉连贯性

### 4.4 可靠性需求
- **数据持久化**：使用 IndexedDB 确保数据安全存储，写入操作延迟 ≤ 50ms，读取操作延迟 ≤ 30ms。
- **错误处理**：完善的错误处理和用户提示，所有 API 调用及本地操作均需捕获异常并给出友好提示。
- **缓存机制**：缓存 TEF 分析结果，缓存命中率 ≥ 80%，缓存读取时间 ≤ 100ms，显著提高性能。

## 18. 接口分析

### 18.1 文档概述
本文档基于项目需求规格说明书，对系统**核心业务接口**进行全维度分析，明确接口分层、分类、请求/响应规范、安全、性能及异常处理规则。接口采用 **RESTful 设计规范**，基于 Next.js API Routes 实现服务端接口，对接客户端、AI第三方接口、本地IndexedDB存储，覆盖所有核心功能模块。

### 18.2 接口设计原则
1. **业务对齐**：100%覆盖需求规格说明书的功能/非功能需求
2. **RESTful 规范**：统一请求方法、路径命名、资源操作逻辑
3. **安全优先**：敏感数据加密、身份认证、输入校验
4. **性能达标**：严格遵循响应时间、数据处理、缓存要求
5. **前后端分离**：接口无状态、数据格式统一、兼容多端（桌面/移动端）
6. **可扩展**：支持新增AI模型、数据类型、分析功能

### 18.3 技术架构与接口分层
系统分为**三层接口**，解耦业务、保障稳定性：
1. **客户端本地接口**：IndexedDB 数据读写（前端直接调用，无网络请求）
2. **项目服务端接口**：Next.js API Routes（核心业务处理、数据计算、权限控制）
3. **第三方AI接口**：OpenAI API（自然语言解析、图片识别、智能建议生成）

### 18.4 核心接口分类与详情
按项目功能模块划分为 **6大核心接口组**，共28个核心接口，以下为关键接口详情：

- 认证方式：请求头携带 `Authorization: Bearer {JWT_TOKEN}`
- 数据格式：请求/响应均为 `application/json`
- 图片上传：`multipart/form-data`

### 18.5 接口通用规范
#### 18.5.1 请求参数规范
- 路径参数：小写+短横线，如 `/api/record/diet/2026-04-08`
- 查询参数：分页 `page=1&size=20`、日期 `date=2026-04-08`
- body参数：驼峰命名，必填项标注，敏感字段加密传输

#### 18.5.2 统一响应格式
```json
{
  "code": 200,        // 状态码：200成功，4xx客户端错误，5xx服务端错误
  "message": "success",
  "data": {},         // 业务数据
  "timestamp": 1712345678901,
  "costTime": 120     // 接口耗时(ms)，满足性能监控
}
```

#### 18.5.3 标准错误码
| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 401 | 未授权/Token过期 | 重新登录 |
| 403 | 无权限访问 | 禁止操作 |
| 400 | 参数错误 | 前端校验修正 |
| 429 | AI API调用超限 | 提示重试 |
| 500 | 服务端异常 | 上报日志+友好提示 |

### 18.6 接口安全规范（适配4.2/9.1 需求）
1. **传输安全**：全接口HTTPS加密，防止中间人攻击
2. **身份认证**：私有接口强制JWT认证，无Token直接返回401
3. **数据加密**：
   - 本地IndexedDB数据：AES-256加密存储
   - API密钥/敏感数据：服务端加密，不返回客户端
4. **输入校验**：全接口参数校验，防XSS、SQL注入攻击
5. **权限控制**：用户仅能访问自身健康数据，禁止越权操作

### 18.7 接口性能规范（适配4.1 需求）
严格遵循项目性能要求，核心指标：
1. **响应时间**
   - 日志提交/数据同步：≤1s
   - 基础查询接口：≤2s
   - AI建议/趋势分析：≤3s
   - 长期数据(≥1年)分析：≤5s
2. **数据处理**
   - 支持单用户日均50+条日志存储/检索
   - 图片上传/压缩：≤5s
3. **缓存优化**
   - TEF分析结果缓存，命中率≥80%
   - 缓存读取时间：≤100ms
4. **并发支持**：1000+在线用户无卡顿

### 18.8 接口异常处理机制
1. **AI接口异常**：OpenAI不可用时，自动触发备用方案，返回提示信息
2. **网络异常**：客户端自动重试（最多3次），失败后提示用户
3. **数据异常**：解析失败时，返回错误信息并保留原始数据
4. **本地存储异常**：IndexedDB写入失败时，自动同步到服务端备份
5. **全链路日志**：接口请求、异常、耗时全量记录，支持监控排查
