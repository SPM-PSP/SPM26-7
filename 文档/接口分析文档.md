# 菲特勒斯健康管理系统 接口分析文档
**适配项目**：菲特勒斯——你的个人健康管理专家
**技术栈**：Next.js 15 (App Router)、AI API (OpenAI)、IndexedDB、AES-256加密、JWT认证
**文档版本**：v1.0.0
**更新时间**：2026年4月8日

## 一、文档概述
本文档基于项目需求规格说明书，对系统**核心业务接口**进行全维度分析，明确接口分层、分类、请求/响应规范、安全、性能及异常处理规则。
接口采用 **RESTful 设计规范**，基于 Next.js API Routes 实现服务端接口，对接客户端、AI第三方接口、本地IndexedDB存储，覆盖所有核心功能模块。

## 二、接口设计原则
1. **业务对齐**：100%覆盖需求规格说明书的功能/非功能需求
2. **RESTful 规范**：统一请求方法、路径命名、资源操作逻辑
3. **安全优先**：敏感数据加密、身份认证、输入校验
4. **性能达标**：严格遵循响应时间、数据处理、缓存要求
5. **前后端分离**：接口无状态、数据格式统一、兼容多端（桌面/移动端）
6. **可扩展**：支持新增AI模型、数据类型、分析功能

## 三、技术架构与接口分层
系统分为**三层接口**，解耦业务、保障稳定性：
1. **客户端本地接口**：IndexedDB 数据读写（前端直接调用，无网络请求）
2. **项目服务端接口**：Next.js API Routes（核心业务处理、数据计算、权限控制）
3. **第三方AI接口**：OpenAI API（自然语言解析、图片识别、智能建议生成）

## 四、核心接口分类与详情
按项目功能模块划分为 **6大核心接口组**，共28个核心接口，以下为关键接口详情：

### 接口通用约定
- 请求根路径：`/api`
- 请求方法：GET(查询)、POST(新增)、PUT(修改)、DELETE(删除)
- 认证方式：请求头携带 `Authorization: Bearer {JWT_TOKEN}`
- 数据格式：请求/响应均为 `application/json`
- 图片上传：`multipart/form-data`

## 五、接口通用规范
### 1. 请求参数规范
- 路径参数：小写+短横线，如 `/api/record/diet/2026-04-08`
- 查询参数：分页 `page=1&size=20`、日期 `date=2026-04-08`
-  body参数：驼峰命名，必填项标注，敏感字段加密传输

### 2. 统一响应格式
```json
{
  "code": 200,        // 状态码：200成功，4xx客户端错误，5xx服务端错误
  "message": "success",
  "data": {},         // 业务数据
  "timestamp": 1712345678901,
  "costTime": 120     // 接口耗时(ms)，满足性能监控
}
```

### 3. 标准错误码
| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 401 | 未授权/Token过期 | 重新登录 |
| 403 | 无权限访问 | 禁止操作 |
| 400 | 参数错误 | 前端校验修正 |
| 429 | AI API调用超限 | 提示重试 |
| 500 | 服务端异常 | 上报日志+友好提示 |

## 六、接口安全规范（4.2/9.1 需求）
1. **传输安全**：全接口HTTPS加密，防止中间人攻击
2. **身份认证**：私有接口强制JWT认证，无Token直接返回401
3. **数据加密**：
   - 本地IndexedDB数据：AES-256加密存储
   - API密钥/敏感数据：服务端加密，不返回客户端
4. **输入校验**：全接口参数校验，防XSS、SQL注入攻击
5. **权限控制**：用户仅能访问自身健康数据，禁止越权操作

---

## 七、接口性能规范（4.1 需求）
严格遵循项目性能要求，核心指标：
1. **响应时间**
   - 日志提交/数据同步：≤1s
   - 基础查询接口：≤2s
   - AI建议/趋势分析：≤3s
   - 长期数据(≥1年)分析：≤5s
2. **数据处理**
   - 支持单用户日均50+条日志存储/检索
   - 图片上传/压缩：≤5s
3. **缓存优化**
   - TEF分析结果缓存，命中率≥80%
   - 缓存读取时间：≤100ms
4. **并发支持**：1000+在线用户无卡顿

---

## 八、接口异常处理机制
1. **AI接口异常**：OpenAI不可用时，自动触发备用方案，返回提示信息
2. **网络异常**：客户端自动重试（最多3次），失败后提示用户
3. **数据异常**：解析失败时，返回错误信息并保留原始数据
4. **本地存储异常**：IndexedDB写入失败时，自动同步到服务端备份
5. **全链路日志**：接口请求、异常、耗时全量记录，支持监控排查
